# Contributing to Python Network Traffic Anomaly Detector

Thank you for your interest in contributing to the Python Network Traffic Anomaly Detector.

This project welcomes improvements that strengthen code quality, detection logic, documentation, testing, and defensive security value.

## Ways to Contribute

You can contribute by:

- Reporting bugs
- Suggesting new detection rules
- Improving documentation
- Adding tests
- Improving error handling
- Proposing performance improvements
- Adding new output formats
- Improving CLI usability
- Expanding sample datasets
- Suggesting future integrations

## Before You Start

Before making a major change:

1. Check the existing issues.
2. Open a new issue describing the proposed change.
3. Explain the problem being solved.
4. Describe the expected behavior.
5. Wait for feedback before beginning large changes.

Small fixes such as typo corrections, documentation improvements, or minor test updates may be submitted directly.
